Panduan Instalasi

Pakpost belum memiliki code signing resmi dari Apple atau Microsoft. Ikuti petunjuk di bawah supaya instalasi berjalan lancar.

Install di macOS

Karena Pakpost belum di-notarize Apple, macOS bisa menandai aplikasi sebagai damaged atau memblokir pembukaan. Ini normal untuk aplikasi yang belum memiliki Apple Developer ID.

  1. Download file .dmg sesuai Mac kamu (Intel atau Apple Silicon).
  2. Buka .dmg dan drag Pakpost ke folder Applications.
  3. Saat pertama kali membuka, klik kanan ikon Pakpost di Applications, pilih Open, lalu klik Open lagi.
  4. Kalau muncul pesan "Pakpost is damaged and can't be opened", jalankan perintah berikut di Terminal: 
    xattr -dr com.apple.quarantine /Applications/Pakpost.app
  5. Buka Pakpost kembali dari Applications.

Tips: Untuk pengalaman tanpa peringatan Gatekeeper, diperlukan Apple Developer Program untuk signing + notarisasi.

Install di Windows

Download file .exe dan jalankan installer. Windows Defender atau SmartScreen mungkin menampilkan peringatan karena installer belum memiliki code signing certificate.

  1. Download file Pakpost_<version>_x64_win.exe.
  2. Jalankan installer dan ikuti langkah instalasi.
  3. Kalau muncul peringatan Windows protected your PC, klik More info lalu Run anyway.

Tips: Peringatan SmartScreen akan hilang setelah aplikasi memiliki Windows code signing certificate.

Install di Linux

Pakpost didistribusikan sebagai file .AppImage yang dapat dijalankan langsung tanpa instalasi.

  1. Download file Pakpost_<version>_x64_linux.AppImage.
  2. Buka terminal di folder download, lalu jalankan: 
    chmod +x Pakpost_*.AppImage
./Pakpost_*.AppImage
  3. Kalau ingin shortcut di menu aplikasi, pindahkan AppImage ke ~/.local/bin/ dan buat file .desktop.
